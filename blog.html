1. What is the difference between map(), filter(), forEach() and find() as Array method?

map(): It's one of the array iterators specifically for looping through an array. After completing the loop it returns an array.

filter(): Another array iterator is used for looping through an array. It also returns an array. filter() takes a function as an argument. The purpose is to filter array elements with respective conditions.
const ary = [34,56,78,901,0,5,82,111,1000,90,23,119,19];
const filteredArry = ary.filter((item) => item > 70);
console.log(filteredArry);// Output: [78,82,111,1000,90,119,19]

forEach(): Unique array iterator, It is used only for looping through Array. It doesn't return anything or even a new array. It is an alternative to For Loop built for the array.

find(): Sometimes we need a particular array element. To find the array element there is an iterator named find(). It takes a function and a condition passes into it. Regarding the condition the find() method just returns an array element, Not a new array.

2. What's difference between Local Storage, Session Storage and Cookies?

Local Storage: Storing data to clients or the user's computer. Local Storage is used for this purpose. It's important cause Login info, unregistered users shopping cart history, payment gateway info, etc. should be stored. Usually Local Storage capacity is 10MB. It doesn't need any HTTP request. The location of Local Storage is in the user's Hard Drive. Data will be stored in the particular browser window for a long time. Data will not be cleared until manually deleted from Local Storage.

Session Storage: It is quite similar to Local Storage. The only difference is the data stored in the Session storage will be cleared after closing the Browser. Usually, its size is 5MB.

Cookies: Cookies are sent by the server. It is sent with a response to an HTTP request and then stored in the user's Hard Drive. It stores users' information. If the user returns to the website, the cookie stored in the user's computer is also sent with the request to the server. Then the server parses it to its own language, And it gets the user's information. Cookies are sent in text form. Usually, its size is 5KB. The purposes are Session Management, Personalization, and Tracking. It doesn't care about the user's privacy. Through cookies, companies are selling user information to third-party. Through cookies, it tracks every step of the users on the internet through networks of websites. It may have expiry date.

3. What's Javascript Event Loop?

We know that Javascript is a single-threaded programming language. But it can perform asynchronous tasks. But how? It is the charisma of Event Loop. Function calls are executed from the call stack sequentially.If a call is executed it will be popped from the call stack. Every task should be a sync task. But what happened to the async task? when an async task is called to the call stack, it will not execute from the call stack. The async task will be sent to the corresponding Browser API & Browser API represents a new thread for the execution of an async task. So sync tasks are executed directly from the call stack and async tasks are processed in Browser API. After processing the completion of the async task, a message regarding the async function execution is queued to the Callback Queue or Event Queue. The async task will be here one after another. But still, the async task will not execute. Here Event Loop works for the execution of the async task.

Basically, it checks whether or not the call stack empty means there are no synchronous tasks. Through Event Loop every async call will move to the call stack from Event Queue as first in first out way. Finally from an empty call stack, every async call will execute.

This is how Javascript can execute the Asynchronous function being a Single Threaded Language with the help of Browser API, Callback Queue, and Event Loop.

How quizTimer() and chooseQuiz() functions work

4. QuizTimer()

Handling timer after starting a quiz. It is an arrow function, that takes only one argument as a parameter. A statement is given to the function at the beginning for validation. The statement tells if the function is called with an argument of truthy value(true), this will stop setInterval() using the clearInterval() method. Else, the function is called without argument it will start a dynamic timer using the clearInterval() method. The clearInterval() method takes a callback function as an argument. SetInterval() method calls the function and executes the function after a certain time, here it is 1000 milliseconds. And it constantly runs after every 1000 milliseconds. Inside the callback, some variables are declared and some global variable is used. These are countHTML, covtMin, mod, min, countSec, etc.

  1. countHTML - It defines The DOM Element that will show the timer
  2. covtMin - It contains a Numeric value.
  3. min - contains the value of minute
  4. countSec - contains the value of second

All these variable, expression and statement is used inside quizeTimer() function for showing timer and even stopping the timer

5. ChooseQuiz()

ChooseQuiz() is used to select every quiz. It's an arrow function that has 2 parameters, the first one is 'index' and another is 'givenAns'. 'index' - This means the index of an array element found from the fetched data. The data comes through this path- "./data/quiz.json".

The parameter 'givenAns' is the option of a quiz. Inside the function the 'isExist' variable checks if any answer's id is matching with any quizData's Id or not.

If 'isExist' is a truthy value, one item will be removed and one will be added to an array declared globally named 'answer'.

If 'isExist' is a falsy value, an item

item = {... quizData[index], givenAns}

will be pushed directly to the array (named 'answer').Finally, the displayAnswers() function will be invoked with an argument. And the argument is the 'answer' array.
